The "Why" (Real-Life Context)

A Pay Head is like a "Rule."

  • Basic Pay is a rule based on Attendance (If you don't come, you don't get paid).
  • HRA is a rule based on Calculation (Usually a % of your Basic).
  • PF & ESI are rules based on Law (Statutory deductions sent to the government).

Step 1: Create Earning Pay Heads

These add money to the employee's pocket.

A. BASIC PAY (On Attendance)

  • Navigation: Create → Payroll Masters → Pay Head
  • Pay Head Type: Earnings for Employees
  • Income Type: Fixed
  • Under: Indirect Expenses
  • Calculation Type: On Attendance
  • Attendance Type: Present
  • Calculation Period: Months (Basis: Calendar Period)

B. HRA & DA (Computed Value)

  • Pay Head Type: Earnings for Employees
  • Under: Indirect Expenses
  • Calculation Type: As Computed Value
  • Computation Info: On Specified Formula → Add Pay Head: Basic Pay
  • Slab: Percentage (HRA = 20%, DA = 40%)

C. OVERTIME (Production Based)

  • Calculation Type: On Production
  • Production Type: Overtime (created on Day 3)

Step 2: Create Deduction Pay Heads (Statutory)

These subtract money for savings/taxes.

A. PROVIDENT FUND (PF @ 12%)

  • Pay Head Type: Employee's Statutory Deductions
  • Statutory Pay Type: PF Account (A/c No. 1)
  • Under: Current Liabilities
  • Calculation Type: As Computed Value
  • Computation Info: On Specified Formula → Basic Pay
  • Value: 12%

B. ESIC (@ 3%)

  • Pay Head Type: Employee's Statutory Deductions
  • Statutory Pay Type: Employee State Insurance
  • Under: Current Liabilities
  • Calculation Type: As Computed Value
  • Computation Info: On Specified Formula → Basic Pay + DA
  • Value: 3%

Step 3: Defining Salary Details (The Link)

Once the Pay Heads (Rules) are ready, link them to the Employee.

  1. Go to Alter → Employee → Rajesh Saxena.
  2. Set Define Salary Details to Yes.
  3. Enter components:
    • Basic: ₹20,000
    • Bonus: ₹1,000
    • Overtime: ₹50 (Rate per hour)
    • HRA, DA, PF, ESIC will show "Auto" (computed)

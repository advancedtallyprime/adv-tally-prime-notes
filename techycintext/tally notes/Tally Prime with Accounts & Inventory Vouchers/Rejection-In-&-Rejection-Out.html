1. Introduction

In business, goods are often returned due to damage, defects, or customer dissatisfaction.

VoucherWhen Used
Rejection OutYou return goods to supplier
Rejection InCustomer returns goods to you

2. Real Business Meaning & Use

Rejection Out: Damaged/wrong goods received → return to supplier

Rejection In: Customer returns goods → you accept back

3. How These Vouchers Work in Tally Prime

VoucherPurposeLinked With
Rejection OutReturn to supplierReceipt Note / PO
Rejection InCustomer returnDelivery Note / SO

✓ Auto-adjusts stock
✓ Maintains full traceability

4. Activation in Tally Prime (F11 Features)

F11 → Inventory Features →

  • Use Rejection In/Out Notes → Yes
  • Record inventory in vouchers → Yes
  • Allow order processing → Yes

5. Step-by-Step Voucher Creation

A. Rejection Out (Return to Supplier)

Gateway → Vouchers → Rejection Out
Select Supplier → Item → Qty → Link to Receipt Note → Save

B. Rejection In (Customer Return)

Gateway → Vouchers → Rejection In
Select Customer → Item → Qty → Link to Delivery Note → Save

6. Practical Business Scenarios

Scenario 1: Supplier sends defective goods → Use Rejection Out

Scenario 2: Customer returns after delivery → Use Rejection In
